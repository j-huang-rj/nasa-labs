<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 192.168.255.1 DNSKEY answers → resolver trust-anchor config | Upstream public key material crosses onto the managed host and becomes a local validation root |
| inventory/runtime facts → named include chain | Misrendered resolver includes could accidentally skip static-stub or trust-anchor config |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | T | `ansible/playbooks/dns.yml` trust-anchor fetch | mitigate | Fetch only `DNSKEY` answers from `192.168.255.1`, filter to KSK flag `257`, and fail the play when zero anchors are returned so malformed or empty upstream data never becomes a trust anchor. |
| T-05-01-02 | D | `ansible/playbooks/roles/bind9/templates/named.conf.j2` resolver include chain | mitigate | Render `named.zones.conf` and `named.trust-anchors.conf` explicitly for resolver mode while keeping `named.keys.conf` authoritative-only, so resolver startup cannot omit required files or ingest authoritative-only secrets. |
</threat_model>
