<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| inventory -> role assertions | Untrusted inventory data defines which clients see which view and which zones get rendered |
| wireguard-derived hostvars -> DNS identity facts | Runtime network facts are transformed into zone names and public VPN answers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | I | `templates/named.zones.conf.j2` | mitigate | Enforce `private.order < public.order` in `tasks/assert.yml` and sort views by `order` before rendering so the public `any` view cannot shadow private clients. |
| T-02-02 | T | `tasks/config.yml` runtime-derived facts | mitigate | Compute router/client/ns IPs and zone names from the existing WireGuard-derived octets in `tasks/config.yml`, not from new tracked inventory values, so Phase 2 cannot drift back to stale `lab_id` state. |
</threat_model>
