## Warnings

### WR-01: Missing PHASE START/END markers in config.yml

**File:** `ansible/playbooks/roles/bind9/tasks/config.yml`
**Issue:** The `config.yml` task file lacks `PHASE [config : START]` and `PHASE [config : END]` debug markers. Per AGENTS.md logging convention, phase boundaries in task files must use these markers. Every other bind9 phase file (`setup.yml`, `verify.yml`, `assert.yml`) and the base role's `install.yml` follow this convention. This file is the only outlier.
**Fix:** Add START/END markers:
```yaml
- name: "PHASE [config : START]"
  become: false
  ansible.builtin.debug:
    msg: "Configure bind9"

# ... existing tasks ...

- name: "PHASE [config : END]"
  become: false
  ansible.builtin.debug:
    msg: "Completed config phase"
```

### WR-02: Redundant arithmetic obscures lab ID formula

**File:** `ansible/playbooks/roles/bind9/tasks/config.yml:18`
**Issue:** The `bind9_derived_lab_id` expression contains a mathematically redundant `* 16) // 16` in its second term. For any integer `n`, `(n * 16) // 16 == n`. The full expression:
```
(_bind9_wg_octets[2] | int * 16) + ((_bind9_wg_octets[3] | int // 16) * 16) // 16
```
simplifies to:
```
_bind9_wg_octets[2] | int * 16 + _bind9_wg_octets[3] | int // 16
```
The extra round-trip multiplication and division makes the formula harder to read and verify. If the intent was different (e.g., to incorporate the prefix length), the redundancy masks the error.
**Fix:** Simplify the expression:
```yaml
bind9_derived_lab_id: "{{ (_bind9_wg_octets[2] | int * 16) + (_bind9_wg_octets[3] | int // 16) }}"
```

## Info

### IN-01: Potentially misleading variable name `bind9_vpn_network_base`

**File:** `ansible/playbooks/roles/bind9/tasks/config.yml:16`
**Issue:** `bind9_vpn_network_base` stores only the last octet of the network base (e.g., `48` for `10.100.5.48/28`), not the full network address. The name suggests a complete network base. Compare with `bind9_vpn_network_cidr` on line 17 which stores the full CIDR.
**Fix:** Consider renaming to `bind9_vpn_network_last_octet` or `bind9_vpn_subnet_base_octet` for clarity. Alternatively, store the full network base address for consistency with `bind9_vpn_network_cidr`.

### IN-02: Self-referencing defaults in argument_specs

**File:** `ansible/playbooks/roles/bind9/meta/argument_specs.yml:24-56`
**Issue:** Several `default` values are Jinja2 templates referencing the same variable (e.g., `default: "{{ bind9_packages }}"` for the `bind9_packages` option). This works because `defaults/main.yml` defines all referenced variables, but it's a fragile pattern — if a variable is removed from `defaults/main.yml` but not from `argument_specs.yml`, the self-reference would fail silently or produce undefined behavior.
**Fix:** Either use literal default values matching `defaults/main.yml` (e.g., `default: ["bind", "bind-utils"]`), or document the dependency explicitly. Current pattern works but should be validated if defaults change.
