# Quick Task 260506-i20: Add SELinux setype attributes to bind9 role for proper file contexts - Context

**Gathered:** 2026-05-06
**Status:** Ready for planning

<domain>
## Task Boundary

Add SELinux `setype` attributes to bind9 role file/template tasks for proper file contexts on AlmaLinux with SELinux enforcing.

</domain>

<decisions>
## Implementation Decisions

### setype scope
- Use `setype` on every `file` and `template` task in config.yml. No restorecon handler — Ansible-native and idempotent.

### dynamic zone dir context
- `/var/named/dynamic` gets `named_cache_t` — correct for DDNS (dynamic DNS updates). Named can write zone transfers and journal files to this context.

### the agent's Discretion
- Exact setype values for other paths (named_conf_t for config files/dirs, named_zone_t for zone data dirs).

</decisions>

<specifics>
## Specific Ideas

- SELinux type mapping for BIND on AlmaLinux:
  - `/etc/named.conf` and `/etc/named/` → `named_conf_t`
  - `/var/named/` zone files → `named_zone_t`
  - `/var/named/dynamic/` → `named_cache_t`
- Apply `setype` parameter to `ansible.builtin.file` and `ansible.builtin.template` tasks in `config.yml`
- No restorecon handler needed — setype is idempotent and Ansible-native

</specifics>
